# 一、项目基本框架预添加模块
| 模块                                      | 作用                                                  | 已实现情况 |
|-----------------------------------------|-----------------------------------------------------|-------|
| 配置类（Config）                             | Spring Boot 全局配置（如 WebMvcConfig、跨域、JacksonConfig 等） | 已完成   |
| 基类（BaseEntity/BaseController）           | 实体/控制器基类，统一封装公共字段/方法                                | 已完成   |
| 日期时间序列化/反序列化工具类                         | JacksonConfig + 自定义工具，统一日期时间格式                      | 已完成   |
| 常量类                                     | 全局静态常量，避免魔法字符串                                      | 已完成   |
| 返回类（统一返回对象）                             | 封装 API 统一返回结构                                       | 已完成   |
| 异常类（GlobalExceptionHandler）             | 全局异常捕获与统一返回                                         | 已完成   |
| 全局拦截器（GlobalInterceptor + WebMvcConfig） | 请求日志、token校验、鉴权、耗时统计挂钩                              | 已完成   |







# 二、拦截器预添加功能
| 序号 | 功能模块                  | 描述                                | 建议实现位置                              | 优先级   | 备注                       |
|----|-----------------------|-----------------------------------|-------------------------------------|-------|--------------------------|
| 1  | **请求日志记录**            | 记录请求路径、方法、IP、参数等                  | `preHandle()` + `afterCompletion()` | ⭐⭐⭐⭐  | 可封装成 `RequestLogService` |
| 2  | **Token 校验**          | 校验 `Authorization` 头中的 token 是否有效 | `preHandle()`                       | ⭐⭐⭐⭐⭐ | 登录、权限系统核心功能              |
| 3  | **权限校验（角色/资源）**       | 判断当前用户是否具备访问目标资源的权限               | `preHandle()`                       | ⭐⭐⭐⭐  | 可与 `@Permission` 注解结合    |
| 4  | **请求耗时统计**            | 记录接口执行耗时，方便性能分析                   | `afterCompletion()`                 | ⭐⭐⭐   | 可封装入日志服务                 |
| 5  | **全链路追踪（TraceId）**    | 为每个请求生成唯一 TraceId，贯穿日志            | `preHandle()`                       | ⭐⭐⭐   | 后续可接入 ELK 或 SkyWalking   |
| 6  | **IP 白名单 / 黑名单过滤**    | 拦截非法来源 IP                         | `preHandle()`                       | ⭐⭐    | 适合对外开放的 API              |
| 7  | **签名验证（防篡改）**         | 校验请求签名参数是否合法                      | `preHandle()`                       | ⭐⭐    | 常用于对接第三方系统               |
| 8  | **限流（Rate Limiting）** | 控制接口访问频率                          | `preHandle()`                       | ⭐⭐    | 可集成 Redis + 计数器算法        |
| 9  | **请求上下文存储**           | 将用户信息、请求信息放入 ThreadLocal          | `preHandle()` + `afterCompletion()` | ⭐⭐⭐   | 为 Controller 层提供用户上下文    |
| 10 | **异常日志记录**            | 拦截层级的异常记录（非统一处理层）                 | `afterCompletion()`                 | ⭐     | 仅作补充性记录                  |
| 11 | **调试模式控制**            | 在开发环境打印额外信息                       | 全局开关                                | ⭐     | 可通过配置中心开关控制              |
| 12 | **请求头校验**             | 验证必要的 header 是否存在（如版本号）           | `preHandle()`                       | ⭐⭐    | 避免请求缺少关键参数               |
